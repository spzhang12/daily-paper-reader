## 摘要
在合并数据集上微调目标检测（OD）模型的前提是标注兼容性，然而，对于语义等价的类别，不同数据集往往编码了相互冲突的空间定义。我们提出了一种智能体化标签协调工作流，在训练前利用视觉语言模型来调和异构来源之间的类别语义和边界框粒度。我们以文档布局检测作为一个具有挑战性的案例研究进行评估，因为不同语料库之间的标注标准差异巨大。如果不进行协调，简单的混合数据集微调会降低预训练 RT-DETRv2 检测器的性能：在衡量完整文档转换流水线还原真实结构准确度的 SCORE-Bench 上，表格 TEDS 从 0.800 下降到 0.750。应用于两个分别具有 16 个和 10 个类别分类体系（且仅有 8 个直接对应关系）的语料库时，协调在内容保真度、表格结构和空间一致性方面均带来了持续提升：检测 F-score 从 0.860 提高到 0.883，表格 TEDS 提高到 0.814，平均边界框重叠度从 0.043 降至 0.016。表示分析进一步表明，协调后的训练产生了更紧凑且更具可分性的解码器后嵌入，证实了标注不一致会扭曲学习到的特征空间，而在训练前解决这一问题可以恢复表示结构。

## Abstract
Fine-tuning object detection (OD) models on combined datasets assumes annotation compatibility, yet datasets often encode conflicting spatial definitions for semantically equivalent categories. We propose an agentic label harmonization workflow that uses a vision-language model to reconcile both category semantics and bounding box granularity across heterogeneous sources before training. We evaluate on document layout detection as a challenging case study, where annotation standards vary widely across corpora. Without harmonization, naïve mixed-dataset fine-tuning degrades a pretrained RT-DETRv2 detector: on SCORE-Bench, which measures how accurately the full document conversion pipeline reproduces ground-truth structure, table TEDS drops from 0.800 to 0.750. Applied to two corpora whose 16 and 10 category taxonomies share only 8 direct correspondences, harmonization yields consistent gains across content fidelity, table structure, and spatial consistency: detection F-score improves from 0.860 to 0.883, table TEDS improves to 0.814, and mean bounding box overlap drops from 0.043 to 0.016. Representation analysis further shows that harmonized training produces more compact and separable post-decoder embeddings, confirming that annotation inconsistency distorts the learned feature space and that resolving it before training restores representation structure.

### 1. 核心问题与整体含义
*   **研究动机**：在目标检测（OD）领域，联合多个来源的数据集进行微调是提升模型泛化能力的常用手段。然而，不同数据集对同一语义类别（如“段落”或“表格”）的标注标准往往存在冲突（例如：有的数据集包含周围空白，有的紧贴文字边缘）。
*   **核心问题**：这种“标注不一致性”会导致模型在训练过程中接收到冲突的监督信号，从而扭曲模型内部的特征表示空间，导致定位不稳定、召回率下降，甚至在简单混合数据训练时性能反而不如单数据集预训练模型。

### 2. 方法论
*   **核心思想**：提出一种**智能体化标签协调（Agentic Harmonization）**工作流。在模型训练之前，利用视觉语言模型（VLM）作为智能体，根据统一的目标标准对异构数据集的标注进行重写和对齐。
*   **关键技术细节**：
    *   **规则驱动推理**：定义一套结构化规则集（Rule Set），规定目标分类体系和空间标注粒度。
    *   **视觉接地推理**：VLM Agent 同时接收原始图像和原始标注，结合视觉上下文判断如何调整边界框。
    *   **约束转换**：采用“保守转换”策略，Agent 可以对原始框进行保留、合并或边界微调（Snap to visual edges），但目前版本不支持将一个框拆分为多个（No-splitting），确保标注转换的可追溯性。
    *   **统一监督信号**：通过公式 $\hat{\mathcal{A}}_i = \mathcal{F}(I_i, \mathcal{A}_i; \mathcal{R})$ 将所有源数据转化为符合目标标准 $\mathcal{R}$ 的一致性标注。

### 3. 实验设计
*   **数据集**：
    *   **Unstructured 内部数据集**：约 4.7 万张图像，16 个类别。
    *   **DocLayNet 25k**：公开数据集的子集，10 个类别。
    *   注：两者仅有 8 个类别直接对应，且空间尺寸差异巨大（如 Unstructured 的表格标注面积平均是 DocLayNet 的 5.2 倍）。
*   **Benchmark**：使用 **SCORE-Bench** 框架，这是一种端到端的语义评估框架，衡量文档转换流水线在内容保真度、表格结构还原、空间一致性等方面的综合表现。
*   **对比方法**：
    1.  **Pretrained (Heron)**：未经额外微调的预训练 Docling Heron 模型。
    2.  **Naïve Fine-tuning**：直接将两个标注不一致的数据集混合进行微调。
    3.  **Harmonized Fine-tuning**：使用本文提出的协调工作流处理后再进行微调。

### 4. 资源与算力
*   **硬件支持**：使用了 **2 张 NVIDIA A100 80GB GPU**。
*   **训练细节**：微调 50 个 epoch，采用 AdamW 优化器，基础学习率 $3 \times 10^{-4}$，Backbone 学习率 $3 \times 10^{-5}$，包含 2000 步预热（Warmup）。

### 5. 实验数量与充分性
*   **实验规模**：针对 17 项端到端指标进行了详细对比，并对 17 个细分预测类别进行了表示空间分析。
*   **充分性与客观性**：
    *   实验不仅对比了最终的检测指标（F-score），还引入了**表示分析（Representation Analysis）**，通过 UMAP 可视化、轮廓系数（Silhouette Score）和邻域纯度（Neighborhood Purity）来量化特征空间的质量。
    *   使用了 SCORE 框架而非传统的 mAP，有效规避了因分类体系不一致导致 mAP 无法公平计算的问题，实验设计较为客观。

### 6. 主要结论与发现
*   **性能提升**：协调后的模型在 14/17 项指标上表现最优。检测 F-score 从 0.860 提升至 0.883；表格结构准确度（TEDS）从 0.800 提升至 0.814。
*   **负面影响验证**：实验证实，如果不进行协调，直接混合数据（Naïve）会导致性能下降（如表格 TEDS 从 0.800 降至 0.750），证明了标注冲突的危害。
*   **特征空间优化**：表示分析显示，协调训练使解码器后的嵌入向量（Post-decoder embeddings）形成了更紧凑、边界更清晰的聚类，减少了易混淆类别（如段落与列表项）之间的重叠。

### 7. 优点
*   **解决本质痛点**：直接针对多源数据集成中被长期忽视的“标注哲学冲突”问题。
*   **表示层面的洞察**：通过深入分析模型内部特征空间的几何结构，证明了标注一致性对模型“理解”布局语义的重要性。
*   **端到端验证**：不局限于检测框的重叠度，而是通过下游文档转换任务的质量来验证实际价值。

### 8. 不足与局限
*   **转换模式受限**：目前不支持“一对多”拆分（One-to-many splitting），无法处理将复合区域（如标题+正文被标为一个框）拆解的情况。
*   **计算成本**：在训练前使用 VLM 对数万张图像进行推理和标注重写，会引入额外的预处理时间和计算开销。
*   **泛化范围**：实验主要集中在文档布局领域，尚未在更通用的自然场景目标检测数据集上验证该方法的普适性。
