## 摘要
部署在跨组织边界的检索增强生成（RAG）系统在安全性、准确性和效率之间面临着根本性的权衡。当前的加密方法在解密过程中会暴露明文，而联邦架构则阻碍了资源整合并产生了巨大的开销。我们提出了 Trans-RAG，它实现了一种新颖的向量空间语言范式，使每个组织的知识都存在于数学隔离的语义空间中。其核心是 vector2Trans，这是一种多阶段变换技术，通过以查询为中心的变换，使查询能够动态地“说出”每个组织的向量空间“语言”，在消除解密开销的同时保持了原生的检索效率。安全评估表明，向量空间接近正交，角间距为 89.90°，隔离率达 99.81%。在 8 个检索器、3 个数据集和 3 个大语言模型（LLM）上的实验表明，准确率下降极小（nDCG@10 下降 3.5%），且相比同态加密在效率上有显著提升。

## Abstract
Retrieval Augmented Generation (RAG) systems deployed across organizational boundaries face fundamental tensions between security, accuracy, and efficiency. Current encryption methods expose plaintext during decryption, while federated architectures prevent resource integration and incur substantial overhead. We introduce Trans-RAG, implementing a novel vector space language paradigm where each organization's knowledge exists in a mathematically isolated semantic space. At the core lies vector2Trans, a multi-stage transformation technique that enables queries to dynamically "speak" each organization's vector space "language" through query-centric transformations, eliminating decryption overhead while maintaining native retrieval efficiency. Security evaluations demonstrate near-orthogonal vector spaces with 89.90° angular separation and 99.81% isolation rates. Experiments across 8 retrievers, 3 datasets, and 3 LLMs show minimal accuracy degradation (3.5% decrease in nDCG@10) and significant efficiency improvements over homomorphic encryption.

在跨组织协作（如医疗研究、供应链管理）中，利用检索增强生成（RAG）共享知识时面临严峻的挑战。现有的安全检索方案存在“不可能三角”：
*   **安全性不足**：传统的文档加密在检索/重排序时需解密，且向量嵌入（Embeddings）本身存在泄露原始文本语义的风险。
*   **效率低下**：同态加密（HE）计算开销巨大，难以满足实时检索需求。
*   **架构冲突**：现有的文档中心化安全范式与现代向量中心化的密集检索架构不匹配。
**Trans-RAG** 的核心动机是打破这一僵局，通过“向量空间语言”范式，使各组织的知识在数学上相互隔离，实现安全、高效且精准的跨组织检索。

#### 2. 论文提出的方法论
论文提出了 **Trans-RAG** 框架，其核心技术是 **vector2Trans**，一种以查询为中心的多阶段向量变换技术。

*   **核心思想**：不加密存储的文档向量，而是为每个组织创建独立的、数学隔离的向量空间。查询在发送至不同组织前，使用该组织特有的密钥进行动态变换，使其“说出”该空间的“语言”。
*   **关键技术细节（vector2Trans 流程）**：
    1.  **基于密钥的置换（Permutation）**：打乱坐标维度，消除维度间的相关性。
    2.  **加密盲化（Cryptographic Blinding）**：添加与输入相关的零均值噪声，干扰统计分析。
    3.  **有界非线性变换（Bounded Non-linearity）**：使用 $f_\beta(x) = \tanh(\beta x)/\beta$ 增加逆向工程难度，同时保留相似性结构。
    4.  **正交旋转与偏移（Orthogonal Rotation & Offsets）**：通过密钥生成的正交矩阵进行旋转，确保变换后的向量模长不变且空间正交。
    5.  **L2 归一化**：确保最终输出在单位超球面上。
*   **整体流程**：查询编码 $\rightarrow$ 查询变换（针对不同组织）$\rightarrow$ 并行检索 $\rightarrow$ 结果聚合与重排 $\rightarrow$ LLM 生成答案。
