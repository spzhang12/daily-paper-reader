## 摘要
AI 智能体正在改变对文档解析的要求。关键在于“语义正确性”：解析输出必须保留自主决策所需的结构和含义，包括正确的表格结构、精确的图表数据、具有语义意义的格式化以及视觉定位。现有的基准测试未能充分涵盖企业自动化中的这一场景，它们往往依赖于狭窄的文档分布和文本相似度指标，从而忽略了对智能体至关重要的失效情况。我们推出了 ParseBench，这是一个包含约 2,000 页经过人工验证的基准测试，涵盖了保险、金融和政府领域的企业文档，并围绕五个能力维度进行组织：表格、图表、内容忠实度、语义格式化和视觉定位。通过对涵盖视觉语言模型、专用文档解析器和 LlamaParse 在内的 14 种方法进行评估，该基准测试揭示了一个碎片化的能力格局：没有任何一种方法能在所有五个维度上都表现出持续的强势。LlamaParse Agentic 获得了最高的综合评分，该基准测试还突显了当前系统仍存在的能力差距。数据集和评估代码已在 HuggingFace 和 GitHub 上发布。

## Abstract
AI agents are changing the requirements for document parsing. What matters is \emph{semantic correctness}: parsed output must preserve the structure and meaning needed for autonomous decisions, including correct table structure, precise chart data, semantically meaningful formatting, and visual grounding. Existing benchmarks do not fully capture this setting for enterprise automation, relying on narrow document distributions and text-similarity metrics that miss agent-critical failures. We introduce \textbf{ParseBench}, a benchmark of ${\sim}2{,}000$ human-verified pages from enterprise documents spanning insurance, finance, and government, organized around five capability dimensions: tables, charts, content faithfulness, semantic formatting, and visual grounding. Across 14 methods spanning vision-language models, specialized document parsers, and LlamaParse, the benchmark reveals a fragmented capability landscape: no method is consistently strong across all five dimensions. LlamaParse Agentic achieves the highest overall score at \agenticoverall\%, and the benchmark highlights the remaining capability gaps across current systems. Dataset and evaluation code are available on \href{https://huggingface.co/datasets/llamaindex/ParseBench}{HuggingFace} and \href{https://github.com/run-llama/ParseBench}{GitHub}.

### 1. 核心问题与研究动机
*   **核心问题**：现有的文档解析基准测试（如 PubTabNet, DocLayNet）主要关注文本相似度或简单的布局检测，无法满足 **AI 智能体（AI Agents）** 对“**语义正确性（Semantic Correctness）**”的高要求。
*   **研究动机**：在企业自动化流程中，解析错误会直接导致决策错误（例如：误读表格表头导致财务分析错误，或忽略删除线导致合同条款理解错误）。因此，解析标准必须从“可供人类阅读”转向“足以支持自主决策”。

### 2. 方法论：核心思想与技术细节
ParseBench 围绕五个关键维度构建，旨在捕捉导致智能体工作流中断的典型失效模式：
*   **表格（Tables）**：引入 **Table Record Match (TRM)** 指标。它将表格视为“记录包”，通过匹配“键-值”对来评分，比传统指标（如 TEDS）更关注表头语义和结构完整性。
*   **图表（Charts）**：采用 **Chart Data Point Match**。不只是生成描述，而是要求提取精确的数据点（数值+标签），并允许一定的视觉估算容差（如 1%）。
*   **内容忠实度（Content Faithfulness）**：通过 14.7 万条自动化规则检测文本遗漏、幻觉和阅读顺序错误。
*   **语义格式化（Semantic Formatting）**：评估对具有语义意义的格式（如**删除线**、**上/下标**、**加粗**、**标题层级**）的保留能力。
*   **视觉定位（Visual Grounding）**：通过 **Element Pass Rate** 综合评估定位（Localization）、分类（Classification）和内容归属（Attribution）的准确性。
*   **标注流程**：采用两阶段流水线：先由前沿视觉语言模型（VLM）生成初始标注，再由人工进行针对性的验证和修正。

*   **对比方法（共 14 种）**：

    *   **专用解析器**：AWS Textract, Azure Doc Intelligence, Google Doc AI, Reducto, Docling (开源) 等。
    *   **LlamaParse**：分为“成本效益版”和“智能体版（Agentic）”。
*   **Benchmark 构成**：总计 16.9 万条测试规则，涵盖 1,180 份文档。

### 4. 资源与算力
*   **算力使用**：论文明确提到，开源模型（Qwen 3 VL, Dots OCR 1.5）和开源解析器（Docling）是在 **NVIDIA H100 GPU** 上运行的（使用 Modal 基础设施）。

### 5. 实验数量与充分性
*   **实验规模**：对 14 种方法在 5 个维度上进行了全面测试，并绘制了“质量-成本”前沿曲线。
*   **充分性**：实验设计非常充分。不仅有宏观的综合评分，还有针对不同难度（简单/中等/困难）的细分分析，以及针对表格指标（TRM vs GriTS）的定性对比。
*   **客观性**：通过统一的 Prompt 模板和标准化的 API 调用，确保了不同供应商之间的对比是公平且可重复的。

### 6. 主要结论与发现
*   **能力格局碎片化**：没有一种方法能在所有五个维度上都表现完美。VLM 在内容理解上较强，但在图表恢复和视觉定位上较弱；专用解析器定位好，但往往忽略语义格式。
*   **LlamaParse 领先**：LlamaParse Agentic 以 **84.9%** 的综合得分位居第一，尤其在语义格式化和视觉定位方面优势明显。
*   **成本与质量的权衡**：增加计算预算（如开启 VLM 的“思考”模式）带来的收益呈边际递减，而专门优化的 Agentic 流水线比单纯增加算力更有效。

### 7. 优点与亮点
*   **贴近实战**：专注于企业级文档（非学术论文），数据分布具有极高的商业参考价值。
*   **指标创新**：TRM 指标解决了传统表格评估对表头不敏感的问题；图表数据点匹配比 QA 形式更严谨。
*   **开源贡献**：数据集和评估框架完全开源，支持社区扩展新的指标和模型。

### 8. 不足与局限
*   **规模限制**：虽然 2,000 页已具规模，但相比于海量的企业文档类型，覆盖范围仍有提升空间。
*   **语言偏差**：目前主要集中在英文文档，对于多语言（尤其是复杂排版的非拉丁语系）的评估较少。
*   **下游任务关联**：目前的评估集中在“解析质量”，尚未直接量化解析错误对具体下游 Agent 任务（如自动化报销、法律合规审查）成功率的具体影响数值。
