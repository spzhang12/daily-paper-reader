## 摘要
如今，金融机构的成功在很大程度上取决于其处理客户生成的数字足迹（例如从各种来源收集的交易历史）以改进用户建模流水线的能力。由于通用大语言模型（LLMs）在处理随时间分布的表格数据方面存在困难，生产技术栈仍依赖于迁移能力有限且需要标注数据的专用表格和序列模型。为解决这一问题，我们提出了 FinTRACE，这是一种检索优先的架构，它将原始交易转换为可重用的特征表示，应用基于规则的检测器，并将生成的信号存储在行为知识库中，该知识库与下游任务的目标具有分级关联。在公开和工业基准测试中，FinTRACE 显著提升了低监督交易分析的效果，将流失预测任务的零样本（zero-shot）马修斯相关系数（MCC）从 0.19 翻倍至 0.38，并将 16 样本（16-shot）MCC 从 0.25 提升至 0.40。我们进一步利用 FinTRACE，通过对检索到的行为模式进行指令微调来增强 LLMs 的知识基础，在交易分析问题上取得了最先进的 LLM 结果。

## Abstract
Nowadays, success of financial organizations heavily depends on their ability to process digital traces generated by their clients, e.g., transaction histories, gathered from various sources to improve user modeling pipelines. As general-purpose LLMs struggle with time-distributed tabular data, production stacks still depend on specialized tabular and sequence models with limited transferability and need for labeled data. To address this, we introduce FinTRACE, a retrieval-first architecture that converts raw transactions into reusable feature representations, applies rule-based detectors, and stores the resulting signals in a behavioral knowledge base with graded associations to the objectives of downstream tasks. Across public and industrial benchmarks, FinTRACE substantially improves low-supervision transaction analytics, doubling zero-shot MCC on churn prediction performance from 0.19 to 0.38 and improving 16-shot MCC from 0.25 to 0.40. We further use FinTRACE to ground LLMs via instruction tuning on retrieved behavioral patterns, achieving state-of-the-art LLM results on transaction analytics problems.

### 2. 方法论：FinTRACE 架构
FinTRACE（金融交易检索与上下文引擎）采用“检索优先”的思路，核心包含以下关键环节：
*   **三层行为知识库 (KB) 构建**：
    1.  **特征精华 (Feature Essences)**：从原始交易序列中提取任务无关的数值描述符（基于 LATTE 框架）。
    2.  **行为模式 (Behavioral Patterns)**：将特征归类为时间动态、货币行为、商户分布三大类，并利用 LLM 总结出高层可解释概念（如“财务稳定性”）。
    3.  **下游目标 (Targets)**：关联具体的预测目标（如流失、违约风险）。
*   **白盒规则提取 (White-box Rules)**：使用 AutoWoE 算法在特征精华与行为模式/目标之间建立显式的、人类可读的逻辑规则（例如：`IF 活跃天数 ≤ 70.5 → 强流失信号`）。
*   **知识增强推理**：
    *   **检索上下文**：在推理时，不再直接输入原始交易行，而是检索并输入与目标相关的行为事实和白盒规则（最多 20 条）。
    *   **指令微调 (Instruction Tuning)**：将 KB 中的三元组转化为指令数据集，使模型内化金融推理模式。
    *   **自反思机制 (Reflection Tuning)**：在少样本场景下，通过存储预测成败的经验，在推理时进行自我修正。

*   **知识复用**：通过构建中间层的行为知识库，使得一套交易数据可以灵活支持多种下游任务，无需为每个新任务重新训练特征提取器。
*   **领域适配**：提出了一种无需人工标注即可进行领域指令微调的方案，降低了金融领域应用 LLM 的门槛。

### 8. 不足与局限
*   **依赖预处理**：FinTRACE 的效果高度依赖于底层特征提取框架（如 LATTE）和规则提取器（AutoWoE）的质量。

*   **实时性挑战**：构建知识库和进行多次 LLM 推理（尤其是带反思机制时）的计算开销较大，可能难以满足超高并发的实时风控场景。
*   **偏差风险**：LLM 引导的行为模式选择可能引入模型固有的偏见，且白盒规则的生成受限于训练数据的分布。
